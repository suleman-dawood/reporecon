<interfaces>
Output contracts the SKILL.md (Plan 06) will rely on:

scripts/preflight.sh
  Exit 0 → prints to stdout: {"core_remaining": N, "search_remaining": M}
  Exit 1 → unauth (stderr: "ERROR: gh CLI not authenticated. Run: gh auth login")
  Exit 2 → gh api rate_limit failed (network/parse)
  Exit 3 → budget too low (core < 50 OR search < 10)

scripts/gh-search.sh <query>
  Exit 0 → prints to stdout JSON array of objects:
    [{full_name, description, stars, pushed_at, archived, language, url}, ...]
  Exit non-zero → query failed; nothing written to stdout

scripts/verify-repo.sh <owner/repo>
  Exit 0 → prints to stdout JSON object:
    {full_name, stars, pushed_at, archived, default_branch, language, url, verified_at, contributor_count}
  Exit 1 → 404 (silent on stdout; caller drops candidate)
  Exit 2 → other gh failure

verified_at = ISO 8601 UTC: $(date -u +%Y-%m-%dT%H:%M:%SZ)

contributor_count is added per Open Question #4 in RESEARCH.md: one extra
`gh api repos/{r}/contributors?per_page=1&anon=true` call, parsed via Link header
trick (jq '.[0]' on contributors array gives 1 contributor; full count requires
HTTP Link header `last` page parsing). Simpler approach: count items in
`gh api repos/${repo}/contributors?per_page=100 --jq 'length'` — accurate for
repos with ≤100 contributors which is the staleness-relevant cohort. If gh fails
this call, output `contributor_count: null` rather than aborting.

  <action>
    Create `scripts/preflight.sh` (mode 0755). Shebang `#!/usr/bin/env bash`,
    `set -euo pipefail`.

    Behavior (exact):
    1. Run `gh auth status >/dev/null 2>&1`. If non-zero, print to stderr
       `ERROR: gh CLI not authenticated. Run: gh auth login` and `exit 1`.
    2. Run `rate_json="$(gh api rate_limit 2>/dev/null)"` — if non-zero, stderr
       `ERROR: gh api rate_limit failed (network? auth?)` and `exit 2`.
    3. Parse with jq:
       core_rem=$(echo "$rate_json" | jq -r '.resources.core.remaining')
       search_rem=$(echo "$rate_json" | jq -r '.resources.search.remaining')
    4. Validate both are integers (regex `^[0-9]+$`); if either is empty or
       non-numeric, stderr `ERROR: could not parse rate_limit JSON. Run \`gh api rate_limit\` manually.` and `exit 2` (fail-closed per RESEARCH.md Open Question 3).
    5. If `core_rem -lt 50 || search_rem -lt 10`, stderr
       `ERROR: gh rate budget too low (core=$core_rem search=$search_rem). Wait for reset.` and `exit 3`.
    6. On success, print to stdout (one line, valid JSON):
       `{"core_remaining":NUMBER,"search_remaining":NUMBER}` and `exit 0`.

    Total length ≤30 lines. No dependencies beyond gh + jq + bash.
  </action>
  <verify>
    <automated>bash -n scripts/preflight.sh && test -x scripts/preflight.sh && grep -q "set -euo pipefail" scripts/preflight.sh && grep -q "gh api rate_limit" scripts/preflight.sh && grep -q "core_remaining" scripts/preflight.sh</automated>
  </verify>

         -f sort=stars \
         -f order=desc \
         -f per_page=10 \
         --jq '[.items[] | {
           full_name,
           description,
           stars: .stargazers_count,
           pushed_at,
           archived,
           language,
           url: .html_url
         }]'
    3. Propagate gh's exit code (set -e handles this).
    4. The output MUST be a JSON array (even if empty `[]`). The `[.items[] | {...}]`
       wrapping with `[ ... ]` ensures this.

    Total length ≤25 lines including a brief header comment citing D-28.
    No additional jq post-processing — output goes directly to stdout for caller.
  </action>

  <action>
    Create `scripts/verify-repo.sh` (mode 0755). Shebang `#!/usr/bin/env bash`,
    `set -euo pipefail`.

    Behavior (exact):
    1. `repo="${1:?usage: verify-repo.sh <owner/repo>}"`.
    2. Fetch repo metadata:
       `if ! repo_json="$(gh api "repos/${repo}" 2>/dev/null)"; then exit 1; fi`
       (exit 1 means 404 or other failure → caller drops candidate).
    3. Fetch contributor count (per Open Question #4 — verify-repo.sh owns this):
       `contrib_count="$(gh api "repos/${repo}/contributors?per_page=100&anon=true" --jq 'length' 2>/dev/null || echo null)"`
       The `|| echo null` ensures we never abort the candidate just because
       contributor enumeration fails (some repos disable it). Output as JSON
       number when integer, JSON null otherwise.
    4. Timestamp:
       `ts="$(date -u +%Y-%m-%dT%H:%M:%SZ)"`
    5. Emit normalized JSON (use jq with --arg + --argjson):
       echo "$repo_json" | jq \
         --arg ts "$ts" \
         --argjson cc "$contrib_count" \
         '{
           full_name,
           stars: .stargazers_count,
           pushed_at,
           archived,
           default_branch,
           language,
           url: .html_url,
           verified_at: $ts,
           contributor_count: $cc
         }'
       (If $contrib_count is `null` literal string, jq --argjson parses null;
       if it's an integer, jq parses as number.)

    Total length ≤35 lines.
  </action>
  <verify>
    <automated>bash -n scripts/verify-repo.sh && test -x scripts/verify-repo.sh && grep -q 'gh api "repos/' scripts/verify-repo.sh && grep -q "verified_at" scripts/verify-repo.sh && grep -q "contributor_count" scripts/verify-repo.sh</automated>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | gh token leak via verbose error | mitigate | All gh calls use `2>/dev/null` to suppress; we never echo $GH_TOKEN or auth headers |
| T-02-02 | Tampering | $1 contains shell metacharacters (`;`, backticks) | mitigate | gh api uses `-f q=` (form parameter) and `repos/${repo}` is quoted; bash `set -euo pipefail` aborts on expansion errors |
| T-02-03 | Denial of Service | Hostile query that triggers gh secondary rate limit | mitigate | preflight.sh aborts run if budget low; Plan 06 SKILL.md adds 200-500ms sleeps between 5 calls per Pitfall 6 mitigation |
| T-02-04 | Information Disclosure | Private repos surfaced in search results | accept | gh search/repositories default excludes private repos; verified explicitly in Task 2 (no `is:private` qualifier appears) |
| T-02-05 | Spoofing | Hallucinated repo cited without verification | mitigate | verify-repo.sh exit 1 on 404 = caller (SKILL.md) MUST drop candidate. Hard rule enforced in Plan 06 |
</threat_model>

| sharpened text → slug | Slug regex strict-allowlists `[a-z0-9-]` — prevents path traversal |
| candidate metadata → report | Only verified candidates reach the report (HARD RULE) |

| T-03-02 | Information Disclosure | gh token / env vars leaking into report | mitigate | "Output Discipline" section forbids token-shaped strings; Plan 06 SKILL.md enforces |
| T-03-03 | Spoofing | Hallucinated URL appearing in report | mitigate | Hard rule: "Never list a URL without a verified_at timestamp" — Plan 06 enforces by passing only verified candidates to template |
| T-03-04 | Tampering | User-supplied proper noun contains shell metacharacters | accept | Sharpening output is consumed by LLM not shell; only slug derivation runs in bash, and the regex strip removes everything dangerous |

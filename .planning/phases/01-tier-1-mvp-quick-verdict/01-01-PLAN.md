| Boundary | Description |

| user→marketplace install | User runs `/plugin marketplace add` pointing at this repo; manifests trusted to declare benign intent |
| user→commands/reporecon.md | $ARGUMENTS is free-form user input — handled downstream by skill, but the command file itself must not interpret it as a path or shell expression |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-01-02 | Information Disclosure | reporecon-reports/ committed by accident | mitigate | .gitignore excludes it (D-23, CLAUDE.md constraint) |

| T-01-04 | Spoofing | Plugin name collision with built-in commands | accept | D-04 defers collision check to Phase 3; `/reporecon` is namespaced enough that risk is low |
</threat_model>

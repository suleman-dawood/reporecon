    Create skills/reporecon/references/judge-rubric.md. Required sections in order:

    1. H1 "Tier 1 Judge Rubric"

    2. H2 "Scope: Tier 1 Only" (per JDG-04 cap, D-18)
       Tier 1 judges from README + metadata ONLY. No clones, no source files.
       Therefore Tier 1 verdicts are capped at WORTH_INSPECTING. The full
       5-level taxonomy (EXACT_MATCH / SIGNIFICANT_OVERLAP / PARTIAL_OVERLAP /
       SUPERFICIAL_MATCH / VAPOR) ships in Phase 2 — never emit those labels
       in Tier 1 output.

    3. H2 "The 5 Axes" (per D-16, JDG-01)
       Each axis is scored as an INTEGER 0-3. Higher = stronger match.

       Table with columns: Axis | Question | 0 | 1 | 2 | 3
       Rows (exact names — used by report template):
       - `core_function` — Does the candidate solve the same primary problem?
         0=different problem, 1=adjacent problem, 2=overlapping problem, 3=same problem
       - `target_audience` — Same users? (per Pitfall 2: score this LAST to resist self-deception)
         0=different users, 1=adjacent users, 2=overlapping users, 3=same users
       - `scope` — Same breadth of features?
         0=much smaller/larger, 1=somewhat different, 2=mostly comparable, 3=same scope
       - `approach` — Same implementation strategy / interface (CLI vs web, framework, etc)?
         0=very different, 1=somewhat different, 2=similar, 3=same approach
       - `activity` — Is the candidate alive? (uses pushed_at, archived, contributor_count)
         0=archived or pushed >18mo ago, 1=stale (>12mo), 2=somewhat active, 3=actively maintained
       Note: `activity` is the only axis judged primarily on metadata (mechanical hints OK); the others must cite README content.

    4. H2 "Evidence Requirement (Tier 1)" (per JDG-02)
       Each axis score ≥2 MUST cite one specific phrase, sentence, or claim from
       the candidate's README/description/topics. The cited evidence goes in the
       JSON `rationale` field. NO file paths in Tier 1 (those require clones —
       Phase 2 only). If an axis is scored ≥2 with no README evidence, cap that
       axis at 1.

    5. H2 "Mechanical Verdict Derivation" (per D-18, JDG-03, T1-06)
       Compute `axis_sum = core_function + target_audience + scope + approach + activity`
       (range 0-15) and `core_pair = core_function + target_audience` (range 0-6).

       Threshold table (exact, mechanical — no LLM judgment here):

       | candidate_verdict | Condition |
       |---|---|
       | LIKELY_MATCH | `core_pair ≥ 5` AND `axis_sum ≥ 11` |
       | WORTH_INSPECTING | (`core_pair ≥ 4` AND `axis_sum ≥ 8`) AND NOT LIKELY_MATCH |
       | UNRELATED | all other cases |

       This is computed in SKILL.md (deterministic), NOT asked of the LLM.
       The LLM emits axis scores; SKILL.md emits the verdict.

    6. H2 "Overall Run Verdict" (per D-19, T1-07)
       After all candidates judged:
       - Any LIKELY_MATCH → 🔴 "This exists"
       - Any WORTH_INSPECTING (no LIKELY_MATCH) → 🟡 "Some overlap"
       - All UNRELATED → 🟢 "No close match"

    7. H2 "Staleness Does Not Auto-Downgrade" (per HEUR-03, D-22)
       Staleness badges (archived, stale-12mo, solo-stale-6mo) are emitted by
       scripts/staleness.sh and surfaced next to the candidate URL. They DO
       inform the `activity` axis score. They DO NOT downgrade the overall
       verdict — a stale-but-matching repo still flags as LIKELY_MATCH (the
       "fork it" case is valid signal, not noise).

    8. H2 "Judge Prompt Template" (per D-17, JDG-07)
       Fenced ``` block containing the prompt the LLM receives per candidate.
       The prompt MUST include all of these elements (in any order, but every
       one literal):

       a) Instruction: "Temperature 0. Respond deterministically."
       b) Instruction: "Judge ONE candidate. Do not batch."
       c) The literal anti-novelty line (per JDG-05, specifics section):
          > The user wants their idea to be novel. Resist this. Your job is to find matches, not validate originality.
       d) Inputs (placeholders):
          - SHARPENED_SENTENCE: {{sharpened_sentence}}
          - PRESERVED_TERMS: {{preserved_terms}}
          - CANDIDATE: {{full_name}}, {{description}}, language={{language}},
            stars={{stars}}, pushed_at={{pushed_at}}, archived={{archived}},
            contributor_count={{contributor_count}}
          - CANDIDATE_README_EXCERPT: {{readme_excerpt_first_3000_chars}}
       e) Instruction: "Score each axis 0-3. For any score ≥2, cite a specific
          phrase from the README or description as evidence."
       f) Instruction: "Do NOT include the original user's framing in your
          reasoning. Use only the sharpened sentence + candidate data."
       g) Output schema (exact JSON):
          ```json
          {
            "axis_scores": {
              "core_function": <int 0-3>,
              "target_audience": <int 0-3>,
              "scope": <int 0-3>,
              "approach": <int 0-3>,
              "activity": <int 0-3>
            },
            "rationale": "<≤2 sentences, must include evidence phrases for axes ≥2>"
          }
          ```
       Note: `candidate_verdict` is NOT emitted by the LLM — SKILL.md derives it
       mechanically from axis_scores per the threshold table.

    9. H2 "Devil's-Advocate Re-Judge" (per D-20, JDG-06)
       Trigger condition (exact): the overall run verdict came out 🟢 AND at
       least one candidate has any axis score ≥2.
       For up to 2 such candidates (budget cap), re-run the judge with this
       additional framing prepended:

       > REVERSE FRAMING: Your task on this re-judge is to argue that this
       > candidate IS the user's idea. List the strongest case for a match.
       > Re-score the 5 axes from that posture.

       If the re-judge's `axis_sum` exceeds the original by ≥3 OR the re-judge
       crosses into WORTH_INSPECTING via the threshold table, downgrade the
       overall run verdict 🟢 → 🟡.

    10. H2 "Output Discipline"
        - LLM emits JSON only — no prose preamble
        - If JSON is malformed, SKILL.md retries once; second failure aborts with error
        - Never emit Phase 2 verdict labels in Tier 1 output

    Total length: 100-220 lines.

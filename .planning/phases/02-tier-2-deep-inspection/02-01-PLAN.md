safe-clone.sh contract (from D2-06):
  Usage: `safe-clone.sh <owner/repo> <dest-parent>`
  - Pre-checks size via `gh api /repos/{owner}/{name} --jq .size` (KB). Reject if > 50000.
  - Creates `DEST=$(mktemp -d -t reporecon-XXXXXX)` under /tmp/reporecon/ (ensure parent exists).
  - Exports `GIT_LFS_SKIP_SMUDGE=1`.
  - `timeout --signal=TERM --kill-after=5s 60s git clone --depth 1 --filter=blob:none --single-branch --no-tags --no-local -- "https://github.com/<owner>/<repo>.git" "$DEST"`
  - On success: prints `$DEST` to stdout. Exit 0.
  - `trap 'rm -rf "$DEST"' EXIT INT TERM` — cleanup on any exit path (success too; caller is expected to consume output or re-clone — but per CONTEXT D2-06 the destination IS cleaned on exit). NOTE: callers will read files BEFORE script exits via stdout invocation in a subshell with the dest held alive externally? Re-read CONTEXT.

Re-reading D2-06: "Destination always under `mktemp -d -t reporecon-XXXXXX`, prefixed `/tmp/reporecon/`. `trap 'rm -rf "$DEST"' EXIT INT TERM` ensures cleanup even on signal/error." This means safe-clone.sh holds the temp dir for the caller's duration. Solution: safe-clone.sh prints DEST and the caller is responsible for `rm -rf` after use, OR safe-clone.sh exec-replaces into a "hold" mode. Cleanest: safe-clone.sh creates DEST, clones into it, prints path, exits 0 WITHOUT cleaning (cleanup is the responsibility of SKILL.md Tier 2 protocol via its own trap on the run-level temp tree). The `trap … EXIT INT TERM` runs ONLY when the clone itself fails or is interrupted mid-clone — protecting against half-clones.

Implementation: use a guard variable `CLONE_OK=0`; trap fires `[ "$CLONE_OK" = 1 ] || rm -rf "$DEST"`. Set CLONE_OK=1 on successful `git clone`. This means: success → caller owns cleanup; failure/signal → script cleans the partial clone.

Exit codes (D2-06):
  0 success
  11 oversize (>50000 KB)
  12 timeout (git clone exceeded 60s)
  13 LFS-only repo (no source files after clone — detect post-clone if 0 non-LFS files)
  1 other error (bad args, gh api 404, network)

    Create `scripts/safe-clone.sh`:
      ```bash
      #!/usr/bin/env bash
      set -euo pipefail

      usage() { echo "usage: safe-clone.sh <owner/repo> <dest-parent-ignored>" >&2; exit 1; }
      [ $# -ge 1 ] || usage
      OWNER_REPO="$1"
      [[ "$OWNER_REPO" =~ ^[A-Za-z0-9._-]+/[A-Za-z0-9._-]+$ ]] || { echo "invalid owner/repo: $OWNER_REPO" >&2; exit 1; }

      mkdir -p /tmp/reporecon
      DEST=$(mktemp -d "/tmp/reporecon/reporecon-XXXXXX")
      CLONE_OK=0
      trap '[ "$CLONE_OK" = 1 ] || rm -rf "$DEST"' EXIT INT TERM

      size_kb=$(gh api "repos/${OWNER_REPO}" --jq .size)
      if [ "$size_kb" -gt 50000 ]; then
        echo "oversize: ${OWNER_REPO} is ${size_kb}KB (>50000)" >&2
        exit 11
      fi

      export GIT_LFS_SKIP_SMUDGE=1
      set +e
      timeout --signal=TERM --kill-after=5s 60s \
        git clone --depth 1 --filter=blob:none --single-branch --no-tags -- \
        "https://github.com/${OWNER_REPO}.git" "$DEST"
      rc=$?
      set -e
      if [ $rc -eq 124 ]; then echo "timeout cloning ${OWNER_REPO}" >&2; exit 12; fi
      if [ $rc -ne 0 ]; then echo "clone failed rc=$rc" >&2; exit 1; fi

      # LFS-only detection
      non_lfs_files=$(find "$DEST" -type f -not -path '*/.git/*' | head -1 || true)
      if [ -z "$non_lfs_files" ] && [ -f "$DEST/.gitattributes" ] && grep -q 'filter=lfs' "$DEST/.gitattributes"; then
        echo "LFS-only repo: ${OWNER_REPO}" >&2
        exit 13
      fi

      CLONE_OK=1
      echo "$DEST"
      ```
